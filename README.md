# 萬用加密工具 使用說明

一個以圖形介面操作的檔案加密小工具。支援 **PDF、Word、Excel、PowerPoint、圖片** 等常見檔案格式的密碼保護，所有運算皆在**你自己的電腦上**完成，不會將檔案或密碼上傳到任何伺服器。

## 系統需求

- **作業系統**：Windows / macOS / Linux 皆可執行圖形介面與 PDF、圖片相關功能
- **Python 版本**：建議 Python 3.9 以上
- **Office 原生加密功能**（Word / Excel / PowerPoint）：
  - 僅支援 **Windows**
  - 電腦需安裝**正版 Microsoft Office**（Word / Excel / PowerPoint 桌面版）
  - macOS / Linux 使用者可以正常使用其他功能，但 Office 加密選項會顯示為「未偵測到，需安裝對應套件」

## 安裝步驟

### 1. 確認已安裝 Python

在終端機（Windows 為「命令提示字元」或 PowerShell；macOS/Linux 為「終端機」）輸入：

```bash
python --version
```

若尚未安裝，請先到 [python.org](https://www.python.org/downloads/) 下載安裝（建議勾選 "Add Python to PATH"）。

### 2. 安裝套件

將 `requirements.txt` 與 `萬用加密工具.py` 放在同一個資料夾，然後在該資料夾中執行：

```bash
pip install -r requirements.txt
```

> **Windows 使用者，若需要使用「Office 原生加密」功能**：
> `requirements.txt` 已自動針對 Windows 加入 `pywin32`，一般情況下不需要另外處理。
> 若安裝時發生問題，可手動執行：
> ```bash
> pip install pywin32
> ```

### 3. （可選）建立虛擬環境

若擔心套件版本互相干擾，建議使用虛擬環境：

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

## 啟動程式

在同一個資料夾中執行：

```bash
python 萬用加密工具.py
```

程式啟動後會開啟一個約 1240×830 的視窗（可自行縮放，最小 1040×660）。

> 若畫面顯示「缺少必要套件 customtkinter」，表示上一步的安裝未成功，請重新執行 `pip install -r requirements.txt`。

## 功能一：加密檔案（5 步驟精靈）

點選左側「🔒 加密檔案」，依照畫面上方的步驟指示條完成以下 5 步：

### 步驟 ① 選擇檔案

- 點「選擇檔案」可多選單一或多個檔案
- 點「📁 選擇資料夾」可整個資料夾一次選取（會列出資料夾內所有可辨識的檔案）
- 已選擇的檔案會列表顯示，並依副檔名顯示對應圖示（📕PDF／📘Word／📗Excel／📙PowerPoint／🖼️圖片）

### 步驟 ② 選擇加密方式

- **PDF 原生加密**：適用 PDF 檔案，任何系統皆可使用
- **Office 原生加密**：適用 Word/Excel/PowerPoint，僅 Windows + 已安裝正版 Office 才能選用

若你選擇的檔案類型與加密方式不符，程式會提示或該選項會被停用。

### 步驟 ③ 設定密碼

- 可自行輸入兩次密碼（需輸入一致才能繼續）
- 也可以點擊「產生高強度密碼」按鈕，自動產生 16 碼、包含大小寫字母／數字／符號的隨機密碼
- 輸入密碼時，畫面會即時顯示**密碼強度分數**與建議（太短 / 弱 / 中等 / 強 / 非常強）

### 步驟 ④ 選擇輸出位置

- **存在原始檔案所在的資料夾（預設，推薦）**：加密後的檔案會與原檔放在同一資料夾
- **另外指定一個資料夾**：點「📂 選擇資料夾」自訂輸出位置

### 步驟 ⑤ 開始加密

- 確認以上設定無誤後，按下「開始加密」
- 畫面下方會顯示即時進度條與處理紀錄（成功／失敗訊息）
- 全部處理完成後，程式會自動開啟輸出資料夾，方便立即查看加密後的檔案

> 💡 已完成的步驟圖示可以直接點擊，快速跳回前面步驟修改設定；但無法跳過尚未完成的步驟。

## 功能二：圖片轉加密 PDF

點選左側「🖼️ 圖片轉加密PDF」：

1. 選擇多張圖片（支援 jpg、jpeg、png、bmp、tif、tiff、gif、webp）
2. 程式會依你選擇的順序，將圖片依序合併成一份 PDF（非 RGB 格式的圖片會自動轉換為 RGB）
3. 設定密碼（同步驟③的密碼強度提示與自動產生功能）
4. 選擇輸出位置與檔名
5. 執行後會產生一份**已加密**的 PDF 檔，內容為你所選圖片依序排列的頁面

適合用於：合併多張證件照、掃描文件、收據照片，並直接加上密碼保護後再分享。

## 功能三：密碼工具

點選左側「🔑 密碼工具」：

- **密碼強度檢測**：輸入任意密碼，即時顯示強度分數（0–100）與文字建議
  - 依「長度」與「字元種類（大寫／小寫／數字／符號）」計算熵值來評分
  - 少於 8 碼會直接判定「太短，容易被破解」
- **密碼產生器**：可自訂長度，並勾選是否包含大寫、小寫、數字、符號，一鍵產生符合條件的隨機密碼（使用 Python `secrets` 模組產生，具備密碼學安全等級的隨機性）

## 兩種加密方式比較

| 項目 | 📕 PDF 原生加密 | 📘 Office 原生加密 |
|---|---|---|
| 開啟方式 | 任何 PDF 閱讀器雙擊開啟即跳出密碼視窗 | Word/Excel/PowerPoint 開啟即要求密碼 |
| 額外工具 | 不需要 | 不需要 |
| 適用檔案 | 僅 PDF | Word / Excel / PowerPoint |
| 系統限制 | 無 | 僅 Windows |
| 額外需求 | 無（已內建） | 需安裝正版 Microsoft Office 桌面版 |
| 加密演算法 | AES-256 | 由 Office 應用程式本身處理 |

程式的「使用說明」頁面會依你目前的環境，自動偵測並顯示「🟢 目前可用」或「⚪ 目前未偵測到，需安裝對應套件」。

## 密碼安全建議

- 建議使用 **12 碼以上**、混合大小寫字母、數字、符號的密碼
- **密碼一旦遺失，沒有任何方式可以救回加密檔案內容**，請務必妥善保存（例如使用密碼管理工具）
- 本程式所有運算都在你自己的電腦上完成，**不會**把檔案或密碼上傳到網路
- 分享加密檔案與密碼時，建議透過不同管道傳送（例如檔案用郵件寄出，密碼用電話或即時通訊另外告知），降低同時被攔截的風險

## 常見問題 FAQ

**Q1：這個工具會不會把我的檔案或密碼上傳到網路？**
不會。所有運算都在你自己的電腦上完成，程式沒有任何網路連線行為。

**Q2：為什麼「Word/Excel/PowerPoint 原生加密」選項是灰色、不能選？**
代表你的電腦目前偵測不到已安裝的正版 Office，或作業系統不是 Windows。請改用「PDF 原生加密」，或改用其他方式保護你的檔案。

**Q3：收到加密檔案的人該怎麼打開？**
- PDF 原生加密：用任何 PDF 閱讀器打開，會直接跳出密碼視窗，不需要額外工具。
- Office 原生加密：用 Office 打開會直接要求密碼，不需要額外工具。

**Q4：忘記密碼怎麼辦？**
非常抱歉，加密檔案的密碼**無法透過本工具或其他方式救回**，這是加密機制本身的設計（否則就不安全了）。請務必在加密前妥善記錄密碼。

**Q5：可以批次處理整個資料夾嗎？**
可以。在步驟①選擇檔案時，點擊「📁 選擇資料夾」即可一次選取整個資料夾內的所有可辨識檔案進行批次加密。

## 疑難排解

| 問題 | 可能原因 | 解決方式 |
|---|---|---|
| 開啟程式顯示「缺少必要套件 customtkinter」 | 尚未安裝或安裝失敗 | 執行 `pip install -r requirements.txt` |
| PDF 相關功能無法使用 | 缺少 `pypdf` 或 `cryptography` | 確認已執行 `pip install -r requirements.txt`，必要時單獨執行 `pip install pypdf cryptography` |
| 圖片轉PDF功能無法使用 | 缺少 `Pillow` | 執行 `pip install Pillow` |
| Office 加密選項無法選取 | 非 Windows 系統，或未安裝正版 Office，或未安裝 `pywin32` | Windows 使用者可執行 `pip install pywin32`；macOS/Linux 使用者無法使用此功能，請改用 PDF 原生加密或其他方式 |
| Office 加密過程中跳出錯誤訊息 | Office 應用程式忙碌中、檔案已在其他視窗開啟、檔案損毀等 | 先關閉該檔案在 Word/Excel/PowerPoint 中的所有視窗後重試 |
| Windows 高解析度螢幕文字模糊 | 顯示卡/系統 DPI 設定較特殊 | v5.1 已內建多層 DPI 感知修正機制，一般情況下會自動修正；若仍模糊，可嘗試以系統管理員身分執行，或調整 Windows 顯示設定中的縮放比例 |
| 執行後視窗開啟卻無反應／當掉 | 加密大量檔案或大檔案時處理較久 | 加密作業會在背景執行緒進行，請留意視窗下方的進度條與處理紀錄，耐心等候即可 |

## 檔案結構與核心邏輯說明（進階／開發者參考）

程式為單一 Python 檔案，主要邏輯分區如下：

- **選用套件檢查**：程式啟動時偵測 `customtkinter`、`pypdf`、`Pillow`、`pywin32` 是否可用，並依偵測結果動態顯示/停用對應功能，未安裝的套件不會導致程式崩潰。
- **`EncryptionEngine`（加密引擎）**：與介面完全分離的純邏輯類別，方便日後獨立測試或重複使用：
  - `pdf_encrypt()`：使用 `pypdf` 對 PDF 加上 AES-256 密碼保護
  - `images_to_encrypted_pdf()`：使用 `Pillow` 合併圖片為 PDF，再呼叫 `pdf_encrypt()` 加密
  - `office_encrypt()`：透過 `pywin32` 呼叫本機安裝的 Word/Excel/PowerPoint COM 元件進行加密（僅 Windows）
- **`App`（主要 GUI）**：以 `customtkinter` 打造的側邊導覽 + 5 步驟精靈流程，並使用 `queue.Queue` 搭配背景執行緒（`threading`）處理加密工作，避免長時間運算時視窗介面凍結（無回應）。
- **`_fix_windows_dpi_blur()`**：僅在 Windows 上執行，依序嘗試 PerMonitorV2 → PerMonitor → System → 舊版 API 的 DPI 感知設定，修正高解析度螢幕文字模糊的問題。

若需要修改介面配色、字型或加入新的檔案類型支援，可直接搜尋檔案中的 `THEME` 字典（配色）與 `classify_file()` / `*_EXTENSIONS`（副檔名分類）進行調整。
