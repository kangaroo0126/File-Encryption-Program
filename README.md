# 🔐 萬用加密工具

一個有圖形介面（滑鼠點一點就好，不用打指令）的桌面小程式，
專門用來幫你的檔案（PDF、Word/Excel/PowerPoint、圖片）加上「打開檔案需要輸入密碼」的保護。

這份 README 寫給**完全不熟悉寫程式或電腦技術的人**看，
會盡量把每個步驟講清楚。如果你只是想「趕快用起來」，
可以直接跳到 [快速開始](#-快速開始四步驟) 那一節。

## 這個工具在做什麼？

簡單說，它是一個「檔案加密小幫手」。你選好檔案、設定密碼，按下按鈕，它就會產生一份「需要輸入密碼才能打開」的新檔案，原本的檔案不會被改動。
具體支援以下功能：

| 功能 | 說明 |
|---|---|
| 🔒 **PDF 加密** | 把任何 PDF 檔案加上密碼保護（採用業界標準的 AES-256 加密） |
| 🔒 **Word / Excel / PowerPoint 加密** | 把 `.doc(x)` / `.xls(x)` / `.ppt(x)` 檔案加上密碼保護，效果等同你自己在 Office 裡設定密碼。**僅限 Windows 電腦，且電腦上必須已安裝正版 Microsoft Office** |
| 🖼️ **多張圖片 → 一份加密 PDF** | 把好幾張圖片（例如身分證正反面、收據掃描檔）合併成一份 PDF，並直接加密，很適合用來保護證件掃描檔 |
| 🎲 **密碼產生器** | 一鍵產生高強度、隨機的密碼，可自訂長度與要不要包含大小寫字母／數字／符號 |
| 🧪 **密碼強度檢測** | 輸入一組密碼，程式會告訴你這組密碼夠不夠強 |

全程都在你自己的電腦上執行，**不需要網路、也不會把任何檔案或密碼傳到網路上**。

## 這個工具「不能」做什麼
如果你要加密的是**其他類型的檔案**（例如壓縮檔、影片、純文字檔，或是 Mac 上的 Office 檔案），**目前這版程式沒有對應的功能**，就算你照著 `使用說明.md` 裡的指示安裝了 `pyzipper`，介面上也不會出現「通用高強度加密」這個選項。

## 🚀 快速開始（四步驟）

> 適合「照著做就好，不想看太多說明」的人。想看更細節可跳到下一節。

### 第一步：安裝 Python（第一次使用才需要）

1. 前往 <https://www.python.org/downloads/>，下載安裝檔。
2. 打開安裝檔，**畫面最下方務必勾選「Add Python to PATH」**，再點「Install Now」。
3. 打開「命令提示字元」或「終端機」，輸入：
   ```
   python --version
   ```
   看到類似 `Python 3.12.3` 的文字就代表安裝成功

### 第二步：安裝這個工具需要的套件

在同一個黑色視窗裡輸入：

```
pip install customtkinter pypdf cryptography pillow
```

看到最後出現 `Successfully installed...` 就代表成功了。這四個套件的用途：

| 套件 | 用途 | 是否必要 |
|---|---|---|
| `customtkinter` | 整個圖形介面（視窗、按鈕、卡片外觀） | ✅ 必要，沒裝程式打不開 |
| `pypdf` | PDF 加密的核心引擎 | ✅ 要用「PDF 加密」和「圖片轉加密PDF」都需要 |
| `cryptography` | `pypdf` 做 AES-256 加密時需要它幫忙 | ✅ 同上 |
| `pillow` | 圖片讀取與轉換（給「圖片轉加密PDF」用） | ✅ 要用「圖片轉加密PDF」才需要 |

如果你**只想加密 Word / Excel / PowerPoint**，而且電腦是 **Windows** 並已安裝**正版
Microsoft Office**，另外再裝一個選用套件：

```
pip install pywin32
```

### 第三步：開啟工具

把 `萬用加密工具.py` 放到一個你好記的資料夾（例如桌面），然後：

- **方法 A（最簡單）**：直接對這個檔案雙擊，或滑鼠右鍵「開啟檔案」選 Python。
- **方法 B（如果 A 沒反應）**：打開終端機，輸入（把路徑換成你實際放檔案的位置）：
  ```
  cd 桌面
  python "萬用加密工具.py"
  ```
  > 注意：因為檔名中間有空格，記得整段檔名要用**雙引號**框起來，

### 第四步：開始加密

程式打開後，左側選單點「🔒 加密檔案」，畫面會用 5 個步驟引導你完成：
① 選擇檔案 → ② 加密方式 → ③ 設定密碼 → ④ 輸出位置 → ⑤ 開始加密。
每一步都可以按「‹ 上一步」回去修改。

## 詳細使用教學

打開程式後，左側是導覽選單，一共 5 個頁面：

```
🏠 開始使用      —— 首頁，快速說明操作方式
🔒 加密檔案      —— 主要功能：把檔案加密（5 步驟精靈）
🖼️ 圖片轉加密PDF —— 把多張圖片合併成一份加密 PDF
🔑 密碼工具      —— 密碼產生器 + 密碼強度檢測
📖 使用說明      —— 程式內建的說明頁（含安裝需求、常見問題）
```

左下角有一個「🟢 功能偵測狀態」小面板，會即時顯示：

- **PDF加密**：是否已安裝 `pypdf`（🟢 可用 / ⚪ 未偵測到）
- **圖片處理**：是否已安裝 `pillow`
- **Office原生**：是否偵測到 Windows + 正版 Office + `pywin32`

如果某個項目顯示 ⚪，代表對應功能目前不能用，回頭檢查「第二步：安裝套件」有沒有漏裝。

### 🔒 加密檔案（5 步驟精靈）

**步驟 ① 選擇檔案**
按「📄 選擇檔案」可一次選多個檔案（也可以按多次累加）；
按「📁 選擇資料夾」可以選整個資料夾（程式會處理裡面所有符合條件的檔案）。
選錯了可以按每一列右邊的「✕」移除，或按「🗑️ 清除選擇」全部清空。

**步驟 ② 加密方式**
目前有兩個選項可選（哪個能選、哪個變成灰色不能選，會依你電腦目前安裝的套件而定）：

- 📕 **PDF 原生加密**：只認得 `.pdf` 檔，其他類型會被自動忽略。
  加密後的檔案，收件人用一般 PDF 閱讀器雙擊打開，就會直接跳出密碼視窗。
- 📘 **Office 原生加密**：只認得 `.doc(x)` / `.xls(x)` / `.ppt(x)`，
  且**必須是 Windows 電腦、並已安裝正版 Microsoft Office** 才能使用；
  Mac 或未安裝 Office 的電腦，這個選項會直接變成灰色不能點。

**步驟 ③ 設定密碼**
輸入兩次密碼（確保沒打錯），或直接按「🎲 自動產生高強度密碼」讓程式幫你隨機產生一組。
下方有「密碼強度」計量條，即時顯示你目前輸入的密碼夠不夠強；
如果太弱（例如太短、只有數字），按「下一步」時程式會跳出警告，
問你是不是真的要繼續使用這組弱密碼。

> ⚠️ 如果是程式自動產生的密碼，畫面上會用彈出視窗完整顯示一次，
> **請務必先複製、貼到安全的地方保存好，再繼續下一步**，
> 因為這組密碼只會顯示這一次，之後如果忘記，沒有任何方法可以救回加密後的檔案內容。

**步驟 ④ 輸出位置**
選擇加密完成後的新檔案要存在哪裡：

- **存在原始檔案所在的資料夾（預設）**：加密後的檔案會跟原始檔案放在一起，
  檔名會自動加上「_加密」字樣（例如 `報告.pdf` → `報告_加密.pdf`）。
  **原始檔案不會被覆蓋或刪除**，兩份檔案會同時存在。
- **另外指定一個資料夾**：按「📂 選擇資料夾」自訂一個存放位置，
  所有加密後的檔案都會集中放到這裡。

**步驟 ⑤ 開始加密**
畫面上方會列出一份設定摘要（選了幾個項目、用哪種加密方式、輸出到哪裡），
確認沒問題後按「🔒 開始加密」。加密過程會在下方黑色的紀錄視窗
即時顯示目前處理到哪個檔案（✅ 代表成功，❌ 代表失敗並附上錯誤原因），
也有進度條顯示整體進度。全部完成後，程式會自動幫你打開存放加密檔案的資料夾。

### 🖼️ 圖片轉加密PDF

適合用來保護「掃描件、證件照」這類多張圖片的資料：

1. 按「🖼️ 選擇圖片（可多選）」，可一次選多張圖片，
   合併後的頁面順序會照你選取／加入的順序排列。
2. 設定密碼（一樣要輸入兩次確認）。
3. 按「📄 產生加密 PDF」，程式會跳出存檔視窗，
   讓你決定新的 PDF 檔要存在哪裡、叫什麼名字。
4. 完成後會自動打開存放的資料夾。

### 🔑 密碼工具

這一頁跟「加密檔案」的流程無關，是獨立的小工具，你可以隨時用它來：

- **密碼產生器**：勾選要不要包含「大寫」「小寫」「數字」「符號」，
  拖動滑桿調整密碼長度（8～64 碼），按「🎲 產生密碼」後，
  可以按旁邊的「📋 複製」把密碼複製到剪貼簿，貼到你要用的地方。
- **密碼強度檢測**：把任何一組密碼貼到輸入框，程式會即時告訴你
  這組密碼的強度（太短 / 弱 / 中等 / 強 / 非常強），方便你檢查
  自己平常在用的密碼夠不夠安全。

## 加密後的檔案要怎麼打開？

不需要另外裝「解密工具」，用一般軟體打開、輸入密碼即可：

| 加密方式 | 打開方法 |
|---|---|
| 📕 PDF 原生加密 | 用任何 PDF 閱讀器（例如 Adobe Acrobat Reader、瀏覽器內建的 PDF 檢視器）雙擊打開，會自動跳出密碼輸入視窗 |
| 📘 Office 原生加密 | 用 Word / Excel / PowerPoint 打開，會自動要求輸入密碼，跟你自己在 Office 裡設密碼的效果一樣 |

把加密後的檔案傳給別人時，記得**用另一個管道（例如當面告知、打電話、另一封訊息）
告訴對方密碼**，不要把密碼和加密檔案放在同一封信、同一則訊息裡，
不然加密就失去意義了。

## 密碼安全建議

- 建議使用 **12 碼以上**、混合大小寫字母、數字、符號的密碼，
  或直接用程式內建的「密碼產生器」自動產生。
- 密碼強度才是關鍵：再強的加密演算法，如果密碼是 `123456` 這種容易猜到的密碼，
  保護力都會大幅下降。
- **密碼一旦遺失，沒有任何人（包含這個工具的開發者）可以幫你救回加密後的檔案內容**，
  這是高強度加密技術的正常特性，請務必事先妥善保存密碼
  （例如寫在密碼管理軟體裡，或抄在一個安全的地方）。

## 常見問題排解 FAQ

| 狀況 | 解法 |
|---|---|
| 打開終端機輸入 `python --version` 顯示「找不到指令」 | 回到「第一步」，重新安裝 Python，安裝時務必勾選「Add Python to PATH」 |
| 輸入 `pip install ...` 顯示「找不到指令」 | 改用 `python -m pip install ...` 試試看（Mac 則用 `python3 -m pip install ...`） |
| 雙擊 `.py` 檔案沒有反應 | 改用終端機開啟（見「快速開始」第三步的方法 B），記得檔名有空格要加雙引號 |

| 「Office 原生加密」選項是灰色 | 代表沒偵測到 Windows + 正版 Office + `pywin32`，這在 Mac 上一定不能用；Windows 上請確認已安裝正版 Office，並執行過 `pip install pywin32` |
| 找不到「通用高強度加密」選項 | 這一版程式本來就沒有這個功能（詳見本文開頭「這個工具不能做什麼」章節），不是安裝問題 |
| 忘記密碼 | 非常抱歉，這是無法救回的，請務必事先妥善保存密碼 |
| 收到加密檔案的人不知道怎麼打開 | PDF／Office 檔直接雙擊打開，跳出密碼視窗時輸入密碼即可，不需要安裝任何額外工具 |
| 畫面文字模糊（Windows 高解析度螢幕） | 這一版程式已內建自動修正，理論上不會再發生；若仍模糊，可嘗試以系統管理員身分執行程式 |

## 給有技術背景的人：程式架構說明

如果你（或你請來幫忙的人）懂一點程式，這裡簡單說明程式碼的內部結構，
方便之後維護或擴充：

- **執行環境**：純 Python 3，GUI 框架為 `customtkinter`（架構在 `tkinter` 之上）。
- **`EncryptionEngine` 類別**（第 187 行起）：把「加密邏輯」和「介面邏輯」完全分開，
  方便日後測試或替換，包含三個靜態方法：
  - `pdf_encrypt(input_path, output_path, password)`：用 `pypdf` 的
    `PdfWriter.encrypt(..., algorithm="AES-256")` 對 PDF 加密。
  - `images_to_encrypted_pdf(image_paths, output_pdf_path, password)`：
    用 `PIL.Image` 把多張圖片存成暫存 PDF，再呼叫 `pdf_encrypt` 加密，
    最後刪除暫存檔。
  - `office_encrypt(file_pairs, password, kind)`：透過 `win32com.client`
    遙控本機安裝的 Word / Excel / PowerPoint 應用程式，
    對開啟的文件設定 `.Password` 屬性後另存新檔。
- **`App` 類別**（第 331 行起）：整個圖形介面的主體，用一個左側導覽列 +
  右側內容區的「Shell」架構，五個分頁各自對應一個 `_build_xxx_page()` 方法。
  「加密檔案」頁面採用 5 步驟精靈（wizard）模式實作，狀態存在
  `self.current_step` / `self.max_step_reached` 等變數裡。
- **背景執行緒**：實際加密過程（`_run_encrypt_job`）跑在獨立的
  `threading.Thread` 裡，透過 `queue.Queue`（`self.progress_queue`）
  把進度／log／完成訊息傳回主執行緒更新畫面，避免加密大量檔案時介面卡住。
- **選用套件皆用 `try/except ImportError` 偵測**（第 35～59 行），
  對應的功能會依偵測結果自動啟用／停用（灰色按鈕），
  不會因為缺少選用套件就讓整支程式崩潰。
- **已知落差**：`requirements.txt` 與 `使用說明.md` 都提到 `pyzipper`
  與「通用高強度加密（AES-256 ZIP，支援任何檔案類型）」，
  但目前 `萬用加密工具 (1).py` 全文搜尋不到 `pyzipper` 或任何 ZIP
  相關程式碼，`EncryptionEngine` 也沒有對應方法、介面上也沒有這個選項。
  若要補上這個功能，可以參考 `pyzipper` 官方文件，
  在 `EncryptionEngine` 裡新增一個類似 `generic_zip_encrypt(paths, output_zip_path, password)`
  的方法（用 `pyzipper.AESZipFile` 以 `AES-256` 模式寫入），
  再到 `_build_wizard_step2` 把方法選項加回去、`_run_encrypt_job`
  加上對應分支即可。

## 安全性與隱私

- PDF 加密採用 **AES-256** 演算法，這是目前業界（含銀行、政府機關）
  廣泛採用的高強度加密標準。
- Office 原生加密則是直接呼叫本機安裝的 Office 應用程式做加密，
  安全性等同你自己在 Word / Excel / PowerPoint 裡手動設定密碼。
- 本程式**完全在本機執行，沒有任何網路連線行為**，
  不會把你的檔案內容或密碼上傳到任何伺服器。
- 唯一會離開你電腦的東西，是你自己選擇要傳送出去的「加密後檔案」——
  而這份檔案沒有密碼是打不開的。
